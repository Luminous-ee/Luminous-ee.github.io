关于Markdown那些事

Markdown 是什么

  • Markdown 是一种轻量级标记语言，它允许人们使用易读易写的纯文本格式来编写文档，方便地转换为 XHTML（或者HTML）文档。

  • 由于 Markdown 的轻量化、易读易写特性，并且支持图片，图表、数学公式，许多网站都广泛使用 Markdown 来撰写帮助文档或是用于论坛上发表消息。 如GitHub、CSDN、简书等，甚至还能用于撰写电子书。

    简单来说，它可以免去你繁琐的排版工作，而更加专注于写作。

    详细介绍请看 百度百科

Markdown 用来做什么

  • 编写说明文档，用它写的文档很多技术论坛、平台都能通用；

  • 撰写博客文章，可以使文字、代码都非常清晰易读；

  • Markdown 格式转换方便，可以轻松地将文本转换为 HTML、PDF 等。

    近几年随着相关应用平台的发展，Markdown 以其独到的优势迅速火起来了。

    Markdown的标记很少，常用的也就十个左右，它是一种非常轻量的标记语言，因此深受技术人员的喜爱。

    更多内容可见知乎问答：md文件怎么用？

Markdown 文件怎么打开

如果你只是想阅读一个 .md 文件，又不想下载 Markdown 编辑器，一个好方法是在浏览器中通过插件阅读。

  1. 以 Edge 浏览器为例，在浏览器中搜索并安装 Markdown Viewer

    勾选 “允许访问文件 URL”

  2. 在浏览器地址栏粘贴相应的 .md 本地文件地址，例如：C:\Users\user\Desktop\markdown.md，即可阅读.md文件。

Markdown 编辑器推荐

  1. 平台集成工具：各大论坛、社区平台自带写作工具，你可以直接在知乎、简书、CSDN 等论坛社区创作。

  2. Typora

    Typora 是一款小巧、轻量的 Markdown 编辑器，它的特点是所见即所得，能够对你编辑的内容进行实时预览。 支持多种公式类型，支持 TOC 等标签。可以轻松添加标题、行内代码、内联格式、公式、图像。

  3. Visual Studio Code

    支持几乎所有的语言，拥有它，你就拥有了全世界！用它写 Markdown 文档只是冰山一角。

    VSCode 默认支持 Markdown，只需要安装一个预览插件：Markdown Preview Enhanced 或 Markdown All in One

支持 Markdown 的编辑器太多，有插件、独立软件等各种形式，功能也不完全一致。有的用于基本写作，有的用来写代码，有的只是博客平台配套的编辑器。这里仅列举两款编辑器。

关于 Markdown 笔记插入图片的问题

Markdown 笔记插入图片，基本有以下 3 种方式：

  1. 插入本地图片

    • 使用相对路径

      例如一个名为 1.png 的图片和 .md 文件放在同一目录下，你就可以用这种方式插入图片: ![](1.ipg)

    • 使用绝对路径，不推荐。(如 ![](C:\1.png))

    缺点是不灵活、不好分享，本地图片的路径更改或丢失都会造成 Markdown 文件调不出图。

  2. 使用图床工具

    图床一般是指储存图片的服务器，专门存储图片，上传后就可以获得图片的对外链接。

    比如 聚合图床

    免费图床只需上传图片，就可以生成外链，很方便地插入 Markdown。

    但是很多免费图床存在服务器不稳定、网站倒闭等隐患，往往不能长久存在！

  3. 利用 GitHub 存储图片，在 Markdown 中引用图片链接

    • 将需要用到的图片上传至 git 仓库；

    • 在仓库根目录下建立文件夹，用于存储图片。

    如果你使用 GitHub Pages 搭建了自己的博客，当然这种方式更加方便。
