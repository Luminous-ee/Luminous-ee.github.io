Github+jsDelivr+PicGo=高速+稳定+免费图床！三步搭建！

2021-08-25

一、介绍

用 markdown 写博客，想插入一些图片，这时候就出现了各种各样的问题

于是，我找上了图床

图床

  • 专门用来存放图片，生成外链，可直接复制到markdown笔记中，不仅节省本地服务器空间，加快了图片打开速度，而且再也不用担心我的图片丢失了。
  • 可是，产生了2个问题：
    1. 收费的和免费的，鉴于囊中羞涩，我当然。。。发现不少图床都是免费的。
    2. 稳定持久安全与否，我想免费使用，必定要承担这些风险

图床现状：

  • 微博图床：自从开启了防盗链，凉凉
  • 免费图床：SM.MS、路过图床、小贱贱图床、ImgURL图床、Postimage等等
    • 无需注册，点开即用，永久储存
    • 各有特色，各有限制，各有缺点
  • 大厂存储：七牛云、又拍云、腾讯云COS、阿里云OSS等
    • 容量限制，操作繁琐，实名认证，域名备案
    • 稳定、强大、可靠（有钱都不是事儿）

  • 那么重点来了，使用GitHub仓库创建一个图床，就是本文标题：高速+稳定+免费

    然鹅，要想高速，CDN加速：jsDelivr出场了

jsDelivr

  • jsDelivr 是国外一家优秀的公共 CDN 服务提供商，也是首个「打通中国大陆与海外的免费 CDN 服务」。

CDN：全称是Content Delivery Network，即内容分发网络。

核心理念，就是将内容缓存在终端用户附近

CDN应用广泛，支持多种行业、多种场景内容加速

  • 本篇文章只谈及利用 jsDelivr 加速 Github 仓库的文件。

    我们可以借此搭建一个免费、高速的图床。

图床工具-picGo

  • 图床工具->自动把本地图片上传到图床，并转换成链接的工具，简化操作，提高创作效率

  • PicGo 是一款优秀的图床工具。用 Electron-vue 开发，支持上传七牛云，腾讯云COS，又拍云，GitHub，阿里云OSS，SM.MS，imgur 等7种常用图床。

二、步骤

新建仓库并生成Token


  1. 创建 GitHub 账号，登录。

    GitHub

  2. 创建一个公开仓库，仓库名随意，如 Img ，ImgBed

    下方选择Public

  3. 进入账户设置: Settings->Developer settings->Personal access tokens->生成新的token

    Note做一些描述，勾选上 repo 的相关权限，即可生成一个 Token。如下图：

    注意：生成之后记得把 Token 复制到你的本地，因为一旦关闭网页将不能再看到它。

配置picGo

下载picGo

下载地址：https://github.com/Molunerfinn/PicGo/releases

mac 用户选择 .dmg 下载，windwos 选择 .exe下载

应用文档：https://picgo.github.io/PicGo-Doc/zh/guide/

配置环境变量

确保你安装了 Node.js， 并且版本 >= 8

PicGo的配置文件在不同系统里是不一样的：

  • Windows: %APPDATA%\picgo\data.json

  • Linux: $XDG_CONFIG_HOME/picgo/data.json or ~/.config/picgo/data.json

  • macOS: ~/Library/Application\ Support/picgo/data.json

    右键此电脑->找到高级系统设置->环境变量->系统变量->找到Path->编辑->新建->配置以上路径

    重启picGo使环境变量生效

配置picGo

  • 设定仓库名：<用户名/图床仓库名> 格式

  • 设定分支名：main

  • 设定Token：粘贴之前生成的 Token

  • 指定存储路径：如 photos/，会在仓库下自动创建 photos/ 文件夹，图片储存在此文件夹下

  • 设定自定义域名：https://cdn.jsdelivr.net/gh/用户名/图床仓库名

    比如 https://cdn.jsdelivr.net/gh/Luminous-ee/ImgBed

    在图片上传后，PicGo会按照自定义域名+储存路径+上传的图片名的方式生成访问链接

    因为我们要使用 jsDelivr 加速访问，所以设置为https://cdn.jsdelivr.net/gh/用户名/图床仓库名格式

三、开始使用

  • 拖拽或点击上传图片
  • 提示成功上传到GitHub仓库
  • 复制Markdown外链，插入Markdown笔记中
